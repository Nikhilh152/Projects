Task 1.2) Overview of Methodology : ¶

I will consider the following classifiers for the prediction of the target variable stroke:

  • K-Nearest Neighbor(KNN)
  • Decision Tree
  • Random Forest

The dataset was preprocessed in Phase 1. The preprocessing involved dealing with the missing values, checking for the irrelevant features, dealing with the outliers and trying to understand with visualisation as to which features are important. In this phase 2, before the feature selection is performed the dataset has to be transformed using the following steps:

  1. bin the descriptive feature age
  2. one hot encoding to - age, gender , ever_married , work_type, residence_type, smoking_status
  3. Label encode of the target feature - stroke
  4. Split the dataset into features and target(stroke)
  5. normalise the numerical descriptive features - MinMax(0-1)

The dataset is imbalanced as it has 3223 rceords for no_stroke(target=0) and 114 records for stroke(target=1). So before the feature selection is performed the dataset is split into train and test and the train data is oversampled with the minority class using the below steps:

  1. Split the dataset into train and test (70:30)
  2. To deal with the imbalanced data - Oversample the data to increase the records of the minority class (stroke = 1) in the training data
  3. Find the top features using train data and cross validation by using the filter method - Fscore
  4. Assess the performance of these top 6 features using the test data

After the feature selection is performed we get 6 best features. These 6 best features of the training data will be used for hyperparameter tuning using GridSearchCV and cross validation on each of the three classifiers mentioned above, considering the Roc_auc as the scoring_metric. After the 3 models are tuned we will fit 5 fold cross validation on the test set and perform the paired t-tests to compare each of the models, to check if any performance difference is statistically significant. We also compare the classifiers with respect to the recall score and the confusion matrices on the test data.

    fs_fit = fs.SelectKBest(filter_method , k=num_features)
    fs_fit.fit_transform(D_train, t_train)
    fs_indices = np.argsort(np.nan_to_num(fs_fit.scores_))[::-1][0:num_features]
    
    # check the best 5 features

    feature_importances = fs_fit.scores_[fs_indices]
    
    mean_score = performance_selected_features(fs_indices)
    #print('The acuracy for the num of features',num_features,'is',mean_score)

Task 3: Critique & Limitations: ¶

Strengths:

  • The feature selection was performed correctly. First the feature selection models Fscore and Mutual Info were compared based on the mean test score. Then the Fscore was selected as it had the highest mean test score. Intially the num_features=1 and then the value of num_features is incremented and the corresponding Roc score is calculated. Then incrementing of the num_features was stopped when num_features = 6, because the Roc score started decreasing. So we choose those 6 best features and continue the analysis.

  • The models were compared using the ROC_AUC and Recall score, which helps us to make sure that we choose the model with the highest TPR. Beacuse we would never want to predict a patient with stroke as no stroke(target class = 0).

  • We use cross validation instead of the Hold-out method. To also make sure that the models performance is not based on lucky split of cross validation we also performed paired t-test to check there if there is any statistical significance difference in the performance of the models

Weakness:

  • The age_old feature has the highest feature importance and hence it contributes more to the model. So this is the main reason for overlapping of the lines in the visualisation for Knn and Decision Tree. Example : Incase of KNN, the model predicts same score for a particular value of n_neighbors even though the value of p is changed. Since the whole model is more dependent on the age_old variable this might lead to overfitting.

Task 4.1) Project Summary: ¶

Phase 1:

  • I preprocessed the data by removing the missing values and setting the irrelevant feature 'id' as index.
  • Removed the outliers in the data by excluding the values which were less than (Q1 - 1.5 IQR) and higher than (Q3 + 1.5 IQR).
  • I had to drop the records like gender='other' as it had only one record and as it won't contribute to the model prediction.
  • Used 1-variable, 2-variable and 3-variable plots to get the insights from the data.

Phase 2:

  • I had to bin the descriptive feature age and did one hot encoding for the nominal variables.
  • Had to split the data into features and target and label encoding of the target variable stroke.
  • Normalise the numerical descriptive features using MinMaxScaler()
  • split the dataset into train and test (70:30).
  • I performed the oversampling of the minority class(stroke = 1) in the training data to resolve the imbalance data issue.
  • Found the best features using the feature selection method F-score.
  • Using the best features I fit three models (KNN, Decision Tree and Random Forest) and did hyperparameter tuning using GridSearchCV and cross validation and compared the performance using roc_auc as the metric.
  • Once the models were tuned , cross validation was performed using the tuned models on the test set and performed paired t-test to check for statistical significant difference between the performance of the models.
  • The models were further compared using the Confusion matrix and the Recall score.
